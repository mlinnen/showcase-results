### Key Decision

**Checked-in definition:** A carver appears in the public list only if their `carver_id` appears in prizes or ranked results (1st/2nd/3rd place). Carvers registered but with zero results are filtered out, preventing registration-leak on the public front end.

**Rationale:** Privacy and presentation. The carvers list is a participation roster, not a raw registration dump. Non-competing registrations should not be visible publicly.

### Technical Implementation

- **Service Layer:** `ResultsService::getCheckedInCarverIds($year)` scans all special prizes and division results, extracts unique carver_id values, returns sorted array
- **Template:** Modified `default.php` to filter `$competitors` array, keeping only carvers whose ID is in the checked-in set
- **Data Integrity:** No changes to JSON schema or data sources — filtering happens at render time
- **Edge Cases:** Empty results (empty checked-in set), duplicate IDs (handled by array unique), multi-year consistency
