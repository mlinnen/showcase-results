## Work Completed

### Parameter Validation (HtmlView Layer)

1. **No parameters** → Usage instructions (not an error)
   - Display `<div class="cca-usage">` with example URLs
   - Educational, not a failure state

2. **Non-numeric carver_id** → "Carver ID must be a number."
3. **Non-numeric year** → "Year must be a valid number."
4. **carver_id without year** → "A year is required when looking up by carver ID..."
   - Explains privacy constraint (carver IDs differ between events)
   - Suggests alternatives

5. **name + carver_id** → Name takes precedence, carver_id silently ignored

### Not-Found States (ResultsService)

1. **Name not found (cross-event)** → "No results found for '{name}'. Please check the spelling..."
2. **Name not found in specific year** → "No results found for '{name}' in {year}. They may have competed in a different year."
3. **carver_id not found** → "Carver #{id} was not found in the {year} results."
4. **Registered but zero results** → "{name} is a registered competitor in {year} but has no recorded results."

### Data File Errors

1. **No results-*.json files** → "No competition data is currently available. Please check back later."
2. **Specified year missing** → "No data available for {year}. Available years: {list}."
   - Calls getAvailableYears() to extract years from filenames
   - Regex pattern: `/results-(\d{4})\.json$/`

3. **Malformed JSON** → Silently skipped during cross-event search; single-file mode shows "Data for {year} is temporarily unavailable"

### Security Measures

1. **HTML escaping:** All user input escaped via esc() helper
   - Uses `htmlspecialchars($str, ENT_QUOTES | ENT_HTML5, 'UTF-8')`
   - Prevents XSS attacks

2. **Numeric validation:** Year validated as numeric BEFORE file path construction
   - Prevents path traversal attacks (e.g., `..`, `/etc/passwd`)

3. **Layered validation:** HtmlView validates syntax, ResultsService validates semantics
   - Fail-fast approach in controller layer

### Error Display Format

- **Usage instructions (no params):** `<div class="cca-usage">` with `<h2>`, `<p>`, `<ul>`, `<code>` examples
- **All other errors:** `<div class="cca-error">` with simple `<p>` containing the message
